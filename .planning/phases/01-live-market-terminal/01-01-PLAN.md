<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → API | HTTP requests hit /api/* and non-/api static paths |
| filesystem → static server | StaticFiles serves files from frontend/out |
| DB file → process | sqlite3 reads/writes db/finally.db |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-01-01 | Information Disclosure | StaticFiles mount | medium | mitigate | Mount StaticFiles at `/` AFTER all `/api/*` routers so API paths are never shadowed; serve only the resolved `frontend/out` directory (no `..` traversal — StaticFiles normalizes paths) |
| T-01-02 | Denial of Service | lifespan startup | medium | mitigate | DB init/seed uses `IF NOT EXISTS` / `INSERT OR IGNORE` so a partially-initialized DB cannot wedge startup; missing static dir logs a warning instead of raising |
| T-01-03 | Tampering | db/finally.db path | low | accept | Single-user local app; file perms are the host's responsibility (documented for Phase 5 volume packaging) |
| T-01-SC | Tampering | python package installs | low | accept | No new backend packages added — schema uses stdlib `sqlite3`, static serving uses FastAPI's bundled `StaticFiles`; existing `uv.lock` unchanged |
</threat_model>
