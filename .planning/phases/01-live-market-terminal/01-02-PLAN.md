## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| market source → cache | background task writes prices into shared in-memory state under a lock |
| cache → SSE/REST consumers | serialized payloads (to_dict / get_history) cross to clients |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-01 | Denial of Service | per-ticker history buffers | medium | mitigate | Use a fixed-`maxlen` deque (~600) per ticker so memory is bounded regardless of uptime or tick rate; in-memory only, never persisted per PLAN.md §6 |
| T-02-02 | Tampering | shared cache mutation | low | mitigate | All new mutations occur under the existing `_lock`; get_history returns a copy so callers cannot mutate internal buffers |
| T-02-03 | Information Disclosure | serialized session_change_percent | low | accept | Price/percent data is non-sensitive market data intended for the client |
| T-02-SC | Tampering | package installs | low | accept | No new packages — `collections.deque` is stdlib; `uv.lock` unchanged |
</threat_model>
