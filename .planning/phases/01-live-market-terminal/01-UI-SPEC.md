# Phase 1 — UI Design Contract: Live Market Terminal

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — Tailwind CSS only. No `components.json` found; Next.js is not yet scaffolded (`frontend/` has only `lib/`, per CONTEXT.md D-09). PLAN.md §10 names only "Tailwind CSS for styling with a custom dark theme" — no component library is specified anywhere in the authoritative spec. Decision made on the user's behalf per the shadcn gate's "N" path: this is a bespoke, data-dense Bloomberg-terminal layout built from custom panels + two canvas chart surfaces, not a forms/CRUD surface that benefits from a shadcn primitive set. **Revisit in a later phase** (e.g. Phase 2 trade bar, Phase 4 chat input) if form-heavy surfaces would benefit from Radix-based primitives — flag for `/gsd-ui-phase 2`. |
| Preset | not applicable |
| Component library | none this phase — all components are custom (`<Panel>`, `<WatchlistRow>`, `<ConnectionDot>`, etc.), styled directly with Tailwind utility classes against the tokens below |
| Icon library | none this phase — literal glyph characters only, per locked decision D-03 ("hover-revealed '×' to remove"). Connection status uses a CSS-drawn dot (`border-radius: 50%` div), not an icon. Do not add an icon dependency for Phase 1. |
| Font | **Inter** (UI text: `Inter, ui-sans-serif, system-ui, -apple-system, sans-serif`). All numeric values (prices, change %, cash/total stubs) use `font-variant-numeric: tabular-nums` so digits don't shift width on every ~500ms SSE update — this is load-bearing for a live-flashing terminal. |

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon/glyph gaps, inline padding (sparkline-to-price gap, badge padding) |
| sm | 8px | Compact element spacing (row internal gaps, error-text margin) |
| md | 16px | Default element spacing — panel gutter/gap, outer page padding, panel internal padding |
| lg | 24px | Section padding (not used at panel level in Phase 1; reserved for later phase headers) |
| xl | 32px | Layout gaps (not used in Phase 1's dense grid; reserved) |
| 2xl | 48px | Major section breaks (not used in Phase 1) |
| 3xl | 64px | Page-level spacing (not used in Phase 1) |

Exceptions:
- **44px minimum click/tap target** for the hover-revealed watchlist remove ("×") control, per WCAG 2.5.5. The visible glyph stays small (14px, matches Body line-height) — the 44px target is achieved via padding on the hit area, not by inflating row height.
- **Watchlist row height = 40px** (5 × 8px, composed from the scale — not a true exception, but called out because it deliberately undercuts a "comfortable" 48px+ row for information density, matching the Bloomberg-terminal precedent in PLAN.md §2).

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Label | 12px | 400 | 1.4 |
| Body | 14px | 400 | 1.5 |
| Heading | 18px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Only two weights used across the whole contract: **400 (regular)** and **600 (semibold)** — never introduce a third weight.

Role usage:
- **Label (12/400)** — timestamps, sparkline captions, session-change-% badges, panel-empty helper copy, axis labels on the main chart, "TOTAL VALUE"/"CASH" micro-labels in the header
- **Body (14/400)** — watchlist ticker price, table-style text, panel body copy, input text/placeholder
- **Heading (18/600)** — panel titles ("Watchlist" is implicit via layout, not labeled; explicit titles on placeholder panels), selected-ticker symbol above the main chart, watchlist ticker symbol (semibold for scannability against the 400-weight price beside it)
- **Display (28/600)** — the live price readout above the main chart; header total-value/cash stub figures render in this size (muted color, see Color) so they don't visually jump when Phase 2 makes them live

|------|-------|-------|
| Dominant (60%) | `#0d1117` | Page canvas background, main chart plot background |

| Accent — Blue (10%) | `#209dd7` | Connection-status dot (**connected** state); selected watchlist row (3px left border + `rgba(32,157,215,0.08)` background tint); main detail chart line + area-fill gradient (`rgba(32,157,215,0.35)` → transparent); focus ring on the watchlist add-ticker input |
| Accent — Yellow | `#ecad0a` | Exactly two uses — nothing else: (1) "FinAlly" wordmark text in the header, (2) connection-status dot in the **reconnecting** state |
| Accent — Purple | `#753991` | **Not used anywhere in Phase 1.** Reserved for Phase 2 primary submit buttons (Buy/Sell in the trade bar), per PLAN.md §2. Declared here so Phase 2's UI-SPEC inherits the same token. |
| Destructive / Negative | `#f85149` | Price-flash background on downtick (fades over 500ms), sparkline down-segments, negative `session_change_percent` text, watchlist-add inline validation error text, connection dot **disconnected** state |
| Positive | `#3fb950` | Price-flash background on uptick (fades over 500ms), sparkline up-segments, positive `session_change_percent` text, connection dot **connected**-state pulse accent |
| Neutral | `#8b949e` | Flat/zero-change text and sparkline segments, disabled/placeholder text, chart axis labels, header stub figures until Phase 2 wires real values |

Accent reserved for (explicit, exhaustive):
- **Blue** `#209dd7`: connected dot, selected watchlist row indicator, main chart line/area, input focus ring — 4 elements, no others.
- **Yellow** `#ecad0a`: header wordmark, reconnecting dot — 2 elements, no others.
- **Purple** `#753991`: none this phase (reserved for Phase 2 buttons).

Never use blue/yellow/purple for body text, borders, or as a generic "highlight" — the semantic green/red/gray set owns all price-direction and status communication.

## Phase 1 Surface Notes

*(Supplementary to the template — carries the locked layout/interaction decisions from `01-CONTEXT.md` that the executor needs verbatim.)*

### Layout shell (desktop ≥1280px)

```
┌─────────────────────────────────────────────────────────────────┐
│ HEADER  56px  (bg-panel, border-bottom 1px border-muted)         │
│  FinAlly (yellow wordmark)      ● Connected   TOTAL VALUE  CASH  │
├───────────────┬─────────────────────────────────────┬───────────┤
│ WATCHLIST      │ MAIN CHART (top ~60% of column B)   │ AI CHAT   │
│ 320px fixed    │                                     │ 340px     │
│ full height    │                                     │ fixed,    │
│ scroll-y       ├─────────────────────────────────────┤ collapsi- │
│ (bg-panel,     │ HEATMAP | POSITIONS | P&L CHART      │ ble       │
│ right border)  │ 3 placeholder panels, equal width    │ (placeh.) │
│                │ (bottom ~40% of column B)            │           │
└───────────────┴─────────────────────────────────────┴───────────┘
```

- Column gaps: 16px (`md` token). Outer page padding: 16px.
- Tablet (768–1279px): AI Chat rail collapses by default (toggle in header, collapsed = 0px width, icon-free text toggle "Chat"); the 3-up placeholder row stacks to a single column below the chart.
- All panels: `bg-panel` (#1a1a2e) surface, `border-muted` 1px border, `border-radius: 6px`.

### Header

- Height 56px, `bg-panel`, bottom border 1px `border-muted`.
- Left: "FinAlly" wordmark, Heading-adjacent weight 600, color `accent-yellow`.
- Right (left-to-right order): connection-status dot + text label ("Connected" / "Reconnecting…" / "Disconnected", Label 12px, color matches dot state) — then "TOTAL VALUE" stub — then "CASH" stub.
- Stub figures (D-06: real numbers arrive Phase 2): render in the Display role (28/600) but colored `neutral` gray, value literal `"—"` (em dash), with a Label-role (12px) caption above reading "TOTAL VALUE" / "CASH". Never render `$0.00` (implies computed-but-zero, which is misleading before Phase 2 wires `/api/portfolio`).

### Connection status dot

- 8px circle (CSS `border-radius: 50%`, no icon).
- `connected` → `positive` green, static. `reconnecting` → `accent-yellow`, subtle 1.5s opacity pulse (0.5↔1.0) to signal activity. `disconnected` → `negative` red, static.
- Driven directly by `PriceStreamContext.status` (`frontend/lib/PriceStreamContext.tsx`) — no separate polling.

### Watchlist panel

- Top: add-ticker input, full width minus 16px horizontal padding, height 40px, placeholder "Add ticker (e.g. PYPL)", uppercases as typed, Enter submits to `POST /api/watchlist` (D-04). Focus ring: `accent-blue`. Inline error per Copywriting Contract renders directly below on rejection.
- Rows: 40px height, 12px horizontal padding, one row per tracked ticker (persisted watchlist order).
  - Left: ticker symbol, Heading role but 14px (i.e., 14/600) for scannability against price.
  - Middle: sparkline — TradingView Lightweight Charts, minimal line (no axes/gridlines/crosshair), ~60px × 24px, colored by direction: `positive` (up), `negative` (down), `neutral` (flat). Backfilled from `GET /api/prices/{ticker}/history` on mount (renders populated), extended live via SSE (D-02) — never renders empty-then-appearing.
  - Right: price (Body 14/400, tabular-nums) stacked above `session_change_percent` (Label 12px, colored `positive`/`negative`/`neutral`, format `+1.23%` / `−0.45%`, always signed).
- Hover reveals a right-aligned "×" remove control inside a 44px hit area (visible glyph 14px). Click removes via `DELETE /api/watchlist/{ticker}` — no confirmation (Copywriting Contract).
- Selected row (drives the main chart): 3px `accent-blue` left border + `rgba(32,157,215,0.08)` row background tint. Clicking anywhere on a row except the "×" selects it.
- Price flash: on each SSE update for a ticker, its row briefly gets a background flash — `positive`/`negative` at ~25% opacity, transitioning to transparent over 500ms (`ease-out`). No flash on `flat` direction. If `prefers-reduced-motion: reduce` is set, skip the background transition entirely; only the price/change-% text color communicates direction.

### Main chart panel

- Header row inside panel: selected ticker symbol (Heading 18/600) + live price (Display 28/600, tabular-nums, flashes background on update using the same mechanics as watchlist rows) + `session_change_percent` badge (Label 12px, pill shape, `positive`/`negative` background at low opacity).
- Chart: TradingView Lightweight Charts, **filled-area line series** (per CONTEXT.md discretion, confirmed). Line color `accent-blue`; area gradient `rgba(32,157,215,0.35)` at top fading to transparent. Background transparent (inherits `bg-canvas`). Grid lines: faint `border-muted` at low opacity, or omitted — executor's call, but never full-opacity gridlines (data-dense but uncluttered). Axis text: `neutral` gray, Label size (12px). Crosshair enabled on hover.
- Backfills from `GET /api/prices/{ticker}/history` on selection (renders populated immediately), extends live from SSE (D-02). Brief backfill-in-flight state: centered Label-role "Loading chart…" in `neutral` gray — must be brief given in-memory history is fast to serve.
- Auto-selects the first watchlist ticker on load (confirmed discretion decision).

### Placeholder panels (Heatmap, Positions Table, P&L Chart, AI Chat)

- Same `bg-panel` / `border-muted` / 6px-radius treatment as all other panels — visually consistent with the live panels, not visually "broken" or unstyled.
- Centered content only: Heading (18/600, `neutral`) + Body/Label copy per the Copywriting Contract table above. No icons, no illustrations, no interactive elements (except the AI Assistant panel's non-functional grayed input bar preview).
