<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → watchlist API | untrusted ticker strings in POST body and DELETE/history path params |
| service → market source | watchlist mutations drive add/remove on the shared background task |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-01 | Tampering | POST /api/watchlist ticker | high | mitigate | Server-side validation: uppercase + strict 1-5 ASCII-letter regex before any DB write or source.add_ticker; reject otherwise with 400 |
| T-04-02 | Tampering | path params (DELETE, /prices/{ticker}) | medium | mitigate | Parameterized SQL only (no string interpolation); uppercase+validate the param; unknown ticker yields empty/no-op, never an injection surface |
| T-04-03 | Denial of Service | unbounded watchlist growth | low | mitigate | INSERT OR IGNORE keeps adds idempotent; symbol length capped at 5; tracked-set membership bounded by validated symbols |
| T-04-04 | Information Disclosure | error detail messages | low | accept | Messages describe only symbol-format/priceability, no internal state |
| T-04-SC | Tampering | package installs | low | accept | No new packages — stdlib sqlite3 + existing FastAPI/Pydantic; uv.lock unchanged |
</threat_model>
